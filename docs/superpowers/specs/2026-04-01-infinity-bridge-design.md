# Infinity Bridge Design Spec

**Date**: 2026-04-01
**Status**: Draft

## Overview

Golden Gate Bridge 3D 시뮬레이션을 "Infinity Bridge"로 확장한다. 끝없는 바다 위 금문교를 포르쉐 박스터(소프트탑 오픈카)로 달리며 경치를 즐기는 멍때리기 경험. 기존 시네마틱 모드는 유지하고, 새로운 드라이브 모드를 추가한다.

## Goals

1. 시각적으로 다리가 끝없이 이어져 보일 것
2. 드라이브 모드에서 무한히 직진 주행 가능할 것
3. 반복 구간의 이음새가 체감되지 않을 것 (심리스)
4. 야간 조명이 풍부하고 분위기 있을 것
5. 포르쉐 박스터가 고품질일 것

---

## 1. 모드 시스템

### 1.1 시네마틱 모드 (기존)
- 현재의 카메라 앵글 시나리오대로 자동 재생
- 기존 FlightCamera 기반 유지

### 1.2 드라이브 모드 (신규)
- UI 버튼 또는 키보드로 진입/퇴장
- 포르쉐 박스터에 탑승하여 일정 속도(~80-100km/h)로 직진
- 차량 조향 없음 — 직진 고정, 속도 고정
- 마우스 이동으로 시선(카메라) 자유 회전 (좌/우/위/아래/뒤)
- `V` 키로 1인칭 ↔ 3인칭 시점 토글

#### 1인칭 시점
- 카메라 위치: 운전석 머리 높이 (~도로면 위 1.2m)
- 대시보드, 핸들, 사이드미러가 보임 (차량 모델 내부 메시 활용)
- 시선 회전 범위: 좌우 ±180도, 상하 -30도 ~ +60도

#### 3인칭 시점
- 카메라: 차량 뒤쪽 ~5m, 위쪽 ~2m에서 추적
- 마우스로 차량 주위 오비탈 회전
- 차량 전체 외형이 보임

---

## 2. 무한 다리 시스템

### 2.1 반복 단위
- **구간**: 메인 스팬(1,280m) + 사이드 스팬 1개(343m) = ~1,623m
- **구성요소**: 타워 3개, 메인 케이블, 서스펜더, 데크(트러스/도로면/가로등/난간)
- **제외**: 앵커리지, 톨플라자, Fort Point 아치, 접근 고가도로

### 2.2 타일링 방식
- 동시에 3개의 타일 유지: "이전", "현재", "다음"
- 플레이어가 "현재" 타일의 후반부에 진입하면:
  - "다음" 타일 앞에 새 타일 생성
  - "이전" 타일을 오브젝트 풀로 반환
  - 역할 재배정: 이전→삭제, 현재→이전, 다음→현재, 신규→다음
- **연속 스크롤** 방식 — 순간이동 없음

### 2.3 월드 리센터링
- 플레이어 위치가 원점에서 일정 거리(예: 5,000m) 이상 벗어나면 전체 씬을 원점 방향으로 이동
- 모든 오브젝트(타일, 동적 요소, 카메라)를 동시에 오프셋 적용
- 부동소수점 정밀도 저하 방지

### 2.4 심리스 이음새 요구사항
- 타일 경계에서 데크, 케이블, 서스펜더, 트러스가 **기하학적으로 완벽히 정렬**
- 반복 단위의 시작/끝이 동일한 단면 형상을 가질 것 (타워 중심 기준 분할)
- 조명 상태(가로등, 안개, 앰비언트)가 타일 간 연속
- 동적 오브젝트(새, 배, 차량)는 타일 경계를 인식하지 않고 월드 좌표 기준 스폰/디스폰
- 텍스처 반복 패턴이 인지되지 않도록 데크 텍스처에 미세한 노이즈 변형 적용
- 타일 전환 시 프레임 드롭 없을 것 (오브젝트 풀링 필수)

---

## 3. 시각적 무한감

### 3.1 거리별 렌더링
| 거리 | 렌더링 |
|------|--------|
| 0~1,623m | 현재 타일: 풀 디테일 (모든 파트) |
| 1,623~3,246m | 인접 타일: 풀 디테일 |
| 3,246~6,500m | LOD 타워 실루엣 3~4개 (간소화 메시) |
| 6,500m+ | 안개로 페이드아웃, 렌더링 없음 |

### 3.2 LOD 타워
- 풀 타워 대비 약 1/10 폴리곤의 간소화 메시
- 실루엣과 비율만 유지 (셀, 포탈 등 디테일 생략)
- 메인 케이블의 캐터너리 라인은 유지 (가는 실린더)
- 서스펜더/트러스 생략

### 3.3 안개
- FogExp2 밀도를 조정하여 ~6,500m에서 완전히 사라지도록
- 야간에는 안개 밀도를 살짝 높여 가로등 빛 산란 효과 강조

---

## 4. 포르쉐 박스터

### 4.1 모델
- **소스**: 웹 공개 GLTF/GLB 모델 (Sketchfab 등)
- **차종**: 포르쉐 박스터 (소프트탑 오픈카, 지붕 접힌 상태)
- **대체 허용**: 포르쉐 718 Boxster, 718 Spyder 등 오픈카 포르쉐
- **라이선스**: CC-BY 또는 CC-BY-NC 이상

### 4.2 품질 요구사항
- 고폴리곤 외형 (50k+ 폴리곤 권장)
- PBR 텍스처: baseColor, metalness, roughness, normal map
- 내부 디테일: 대시보드, 핸들, 시트, 센터콘솔 (1인칭에서 보이므로 필수)
- 유리: 투명 + 환경 반사 (MeshPhysicalMaterial, transmission)
- 소프트탑: 접힌 상태의 천 지붕이 뒤쪽에 모델링되어 있을 것
- 휠: 회전 애니메이션 (속도에 비례)

### 4.3 차량 조명
- **헤드라이트**: SpotLight 2개, 도로 전방 비춤, 야간에만 자동 점등
- **테일라이트**: 붉은 PointLight/MeshStandardMaterial(emissive), 야간 상시 점등
- **계기판**: 1인칭에서 보이는 은은한 emissive 발광 (야간)
- **DRL(주간 주행등)**: 주간에도 미약하게 점등

### 4.4 차량 배치
- 6차선 중 우측 2번째 차선 중앙에 고정
- 도로면에서 차체 바닥까지 적절한 오프셋

---

## 5. 동적 환경 요소

### 5.1 새 (BirdSystem 확장)
- 현재 BirdSystem 활용
- 드라이브 모드: 플레이어 전방/측면 200~800m 반경에서 무리 스폰
- 뒤쪽으로 800m 이상 벗어나면 디스폰
- 안개 밖에서만 스폰/디스폰하여 팝인 방지

### 5.2 배
- 간단한 보트/요트 모델 2~3종 (BoxGeometry 기반 또는 간단한 GLTF)
- 수면 위 느리게 이동 (5~15 노트)
- 동시 3~5척 존재
- 야간 항해등: 현측등 (좌현 빨강 PointLight, 우현 녹색 PointLight) + 선미등 (백색)
- 선실 창문에서 따뜻한 emissive 발광

### 5.3 NPC 차량

#### 모델 소싱
- **소스**: 웹 공개 GLTF/GLB 고품질 차량 모델 (Sketchfab 등)
- **브랜드 다양성 필수**: 아우디, 벤츠, BMW, 포르쉐, 테슬라, 도요타, 혼다, 포드 등
- **최소 8~10종 이상**의 서로 다른 차량 모델 확보
- **차종 다양성**: 세단, SUV, 스포츠카, 픽업트럭 등 혼합
- **품질**: PBR 텍스처, 실제 차량과 흡사한 외형 (외부만 — 내부 디테일 불필요)
- **라이선스**: CC-BY 또는 CC-BY-NC 이상

#### 스폰 규칙
- **랜덤 스폰**: 차량 종류, 색상, 속도가 랜덤하게 결정
- **반대 차선**: 맞은편에서 차량이 간헐적으로 다가와 지나감
- **같은 방향**: 가끔 앞서가는 차량 또는 추월하는 차량
- **스폰 간격**: 랜덤 (평균 5~20초, 포아송 분포)
- **안개 밖에서 스폰**, 뒤쪽 안개 밖에서 디스폰 — 자연스러운 등장/퇴장
- 야간 헤드라이트/테일라이트 포함
- 각 차량의 속도에 미세한 편차 (±10km/h) → 자연스러운 흐름

---

## 6. 야간 조명 시스템

### 6.1 다리 조명
- **가로등**: 현재 LightStandards 활성화, 50m 간격, 따뜻한 색온도 (3000K)
- **타워 항공등**: 빨간 점멸 PointLight (현재 AviationLights 활용)
- **타워 업라이트**: 타워 기둥 아래에서 위로 비추는 은은한 SpotLight (야간 전용)

### 6.2 수면 반사
- Water 셰이더에 주변 광원 반사 반영
- 가로등, 차량 라이트, 배 조명이 수면에 길게 비침

### 6.3 Bloom/Glow
- PostFXPipeline의 bloom 강도를 야간에 높여서 조명 산란 효과
- 안개 속 빛이 퍼지는 느낌

---

## 7. 환경

### 7.1 유지
- Water (수면 시뮬레이션)
- SkyController + TimeOfDay (시간대 변화, 일출/일몰/야간)
- WeatherSystem (선택적 — 맑은 날씨 기본)
- PostFXPipeline (bloom, color grading)

### 7.2 제거
- TerrainGenerator의 언덕/해안 지형
- Cityscape (도시 스카이라인)
- Fort Point Arch, 앵커리지, 톨플라자

### 7.3 수정
- 안개 밀도: 무한감에 맞게 조정
- 카메라 far plane: 현재 80,000m 유지 (LOD 타워 렌더링에 충분)

---

## 8. UI/UX

### 8.1 모드 전환
- 메인 화면에 "CINEMATIC" / "DRIVE" 모드 선택 버튼
- 또는 키보드 단축키 (예: `M` 키)

### 8.2 드라이브 모드 HUD
- 속도계 (km/h) — 미니멀 디자인
- 현재 시점 표시 (1st / 3rd)
- 시간대 표시 (기존 Clock.ts 활용)

### 8.3 로딩
- 차량 모델 비동기 로드, 로딩 프로그레스 표시
- 타일 생성은 백그라운드, 프레임 드롭 없이

---

## 9. 기술 제약 및 성능

### 9.1 메모리
- 동시 3타일 + LOD 타워만 메모리에 유지
- 반환된 타일의 geometry/material은 오브젝트 풀에서 재사용 (dispose 하지 않음)
- 차량 모델은 1회 로드 후 재사용

### 9.2 성능 목표
- 60fps 유지 (중간 사양 GPU 기준)
- 타일 전환 시 프레임 스파이크 없음
- LOD 전환이 시각적으로 인지되지 않을 것

### 9.3 부동소수점
- 월드 리센터링으로 Z축 좌표를 주기적으로 0 근처로 리셋
- 리센터링 시 모든 오브젝트 동시 이동 (1프레임 내 완료)

## 10. 파일 구조 (예상)

```
src/
  drive/
    DriveMode.ts          # 드라이브 모드 진입/퇴장, 상태 관리
    InfinityTileManager.ts # 3-타일 관리, 스폰/디스폰, 리센터링
    DriverCamera.ts        # 1인칭/3인칭 카메라 전환, 마우스 시선
    VehicleController.ts   # 박스터 위치/속도/휠 애니메이션
  landmarks/bridge/
    BridgeTile.ts          # 반복 단위 타일 생성 (기존 파트 조합)
    LODTower.ts            # 간소화 타워 메시
  traffic/
    BoatSystem.ts          # 배 스폰/이동/조명
  ui/
    DriveHUD.ts            # 속도계, 시점 표시
```
